# Changelog

本文件记录 Markdown Lite 的用户可见变化。版本号遵循 [Semantic Versioning](https://semver.org/)。

## [Unreleased]

## [0.8.0] - 2026-07-23

### Added

- 工作区会话保留 `current` 与 `previous` 两代槽位；当前会话损坏或缺失时可恢复最近一份有效标签顺序和活动标签。
- 本地恢复草稿同样保留两代；`current` 损坏、缺失或身份不匹配时，只回退到同一文档最近一份有效的 `previous`。

### Changed

- 文档成功保存、重载或明确丢弃会清理两代草稿槽位，过期后台任务不会复活旧草稿；上一代恢复会明确显示，双代失败则保留损坏证据并阻止静默关闭。
- 增加 release 恢复性能门禁：1MB 草稿轮换中位数小于 100ms、回退中位数小于 50ms，100 标签会话回退中位数小于 20ms。
- 两代槽位仅提供一代本地故障回退，不是版本历史、云备份或跨设备同步；没有有效上一代或草稿身份不匹配时无法自动恢复。
- 版本统一为 0.8.0（build 8），继续作为只附源码、不上传本地 ad hoc 应用包的 source-only prerelease。

## [0.7.0] - 2026-07-23

### Added

- 在无序、有序和任务列表行按 `Return` 可延续原标记及缩进；有序编号自动递增，新任务重置为未完成，空列表项会退出列表。
- 增加 1MB 文档的智能 `Return` 性能回归用例；续写计划阶段不额外复制全文或直接调用完整 Markdown 解析器。

### Changed

- 输入法组合、文本选区、普通正文和代码围栏内的 `Return` 保持系统原生行为；本版不增加 `Tab` 缩进、引用延续或全局列表重排。
- CI 从 `Support/Info.plist` 读取并核对应用版本与构建号；版本标签构建另外要求标签名与 `vX.Y.Z` 一致，不再逐版硬编码断言值。
- 版本统一为 0.7.0（build 7），继续作为只附源码、不上传本地 ad hoc 应用包的 source-only prerelease。

## [0.6.0] - 2026-07-23

### Added

- 预览中的任务复选框现在可直接切换 Markdown 原文，并支持格式菜单、`⌘⇧X`、原生撤销重做和 VoiceOver。
- 增加 Bug/Feature Issue Forms、Pull Request 模板、行为准则和 GitHub Actions Dependabot 配置。

### Changed

- 本地图片预览统一限制在已保存文档目录内；单图上限 25 MiB、40 MP，后台生成最长边 1024 px 的缩略图并最多并发解码两张。
- GitHub Actions 只在 Pull Request、`main` 和版本标签运行，`actions/checkout` 固定到经过核验的提交。

### Fixed

- 最近文件索引更新失败时，已经打开的标签仍会进入会话，不再被误报为打开失败。
- 原文件已删除或不可读时，关闭 dirty 标签只允许另存为或取消，避免留下没有恢复入口的孤儿草稿。
- 任务预览写回前会核对解析代次、标签、源行、正文状态和原始 UTF-8 内容，避免旧预览覆盖延迟重载或 Unicode 原文。
- 本地图片读取会拒绝目录外路径、软链接竞态、超限字节和像素，并在预览离屏后取消旧解码任务。

## [0.5.0] - 2026-07-23

### Added

- HTML 导出改为可离线携带的单文件：安全范围内的本地图片会嵌入导出结果，移动 HTML 后仍可查看。

### Changed

- 导出的 HTML 默认不自动加载远程图片，并增加内容安全策略，避免打开导出文件时意外联网；本地图限制为单张 25 MiB、累计 100 MiB。
- 预览解析可真正取消过期任务；大纲同步优先定位目标位置之前最近的内容块。

### Fixed

- 修复重新载入较短文档后旧撤销记录可能越界崩溃，并保持其他标签的撤销历史不受影响。
- 修复撤销或重做回到已保存内容时标签仍显示未保存、旧恢复草稿仍可能复活的问题。
- 文件已写入但最近文件索引更新失败时，不再把整个保存误报为失败。
- 取消或并发导出不会留下临时文件，也不会让较早请求覆盖同一路径的较新结果。

## [0.4.0] - 2026-07-23

### Added

- 新增原生“格式”菜单，集中提供粗体、斜体、行内代码、链接和标题命令。
- 远程图片改为默认不联网：阻止明文 HTTP，HTTPS 仅在逐图确认后加载。
- 将 v0.4.0 定为首个公开 Beta，并补齐真实 GitHub 仓库链接、CI 状态入口、贡献和安全报告说明。

### Changed

- 统一应用版本为 0.4.0、构建号为 4，并明确首个公开版本与早期开发记录的边界。
- 解析器复用列表、引用和段落首行的识别结果；性能门禁从 debug 测试进程校正为独立 release 进程，数值门槛仍为 50ms/200ms。
- 明确 v0.4 预编译验证限于 Apple Silicon（arm64），本地 ad hoc 签名应用不能作为正式下载。
- 保持 CI 聚焦格式、构建、测试、自检和本地应用打包校验，不引入未经验证的自动发布链路。

## [0.3.0] - 2026-07-22

### Added

- 增量 Markdown 语法着色、字号与行距设置。
- 粗体、斜体、行内代码、链接和六级标题快捷格式命令。
- 标准 SwiftPM 测试、统一检查脚本和 GitHub Actions 门禁。
- 外部文件内容指纹检测、冲突状态及明确的重载/覆盖处理入口。

### Changed

- 自动草稿改为后台编码和原子写入，并阻止旧任务复活已清理草稿。
- 草稿持久化编辑起点的磁盘指纹，重启后仍会阻止覆盖进程外新版本。
- 大文档着色限制在可见区域附近，输入期间合并重复刷新任务。

## [0.2.0] - 2026-07-22

### Added

- 多文档标签、标签切换快捷键和上次会话恢复。
- 每个文档独立草稿、预览任务、编码/BOM 和保存状态。
- 本地图片拖放/粘贴、同级 `assets/` 管理与相对路径预览。
- 标题大纲与编辑器/预览跳转。
- “简洁”和“技术文”两种公众号复制模板。
- 开源协作、隐私、安全和 CI 基线。

### Changed

- 长文 Markdown 解析在后台执行，并丢弃过期结果。
- 标签关闭、应用退出和外部文件变化采用更保守的数据保护策略。

## [0.1.0] - 2026-07-22

### Added

- 基于 SwiftUI 与 AppKit 的原生 macOS Markdown 编辑器。
- Markdown 编辑/预览、查找替换、最近文件、另存为和恢复草稿。
- 无损文本编码读取、原子保存及外部修改确认。
- HTML 导出和公众号格式复制。
- 200KB/1MB Markdown 性能自检。
- 本地 `.app` 打包与 SVG 图标生成脚本。
